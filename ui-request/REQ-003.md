# REQ-003: ol/ul ベースリセットに list-style: none を追加

## 概要
- **種別**: CSS Class (Base Reset)
- **優先度**: High
- **依頼元**: openground-ec

## 既存チェック
- **既存コンポーネント**: 該当なし
- **既存CSSクラス**: `.list-disc`, `.list-inside` がユーティリティに存在するが、ベースリセットに `ol, ul` のリセットがない
- **既存トークン**: 該当なし
- **判定**: 既存拡張が必要（`reset.css` への追加）

## 何が必要か
`reset.css` に `ol, ul` の `list-style: none` と `padding: 0` が含まれていないため、`<ol>` を使ったパンくずリストなどでブラウザデフォルトの番号マーカー（1. 2. 3.）が表示されてしまう。Tailwind の Preflight や modern-normalize と同等のリストリセットが必要。

## 仕様

### 修正対象
`src/css/reset.css`

### 追加すべきルール
```css
/* --- Lists --- */
ol, ul {
  list-style: none;
  padding: 0;
}
```

`margin: 0` は既に `*` セレクタでリセット済みのため不要。

### 意図的にリスト表示したい場合
既存のユーティリティクラス `.list-disc` / `.list-inside` や、`.prose` コンテキスト内の `ol li { list-style-type: decimal }` / `ul li { list-style-type: disc }` で明示的にオプトインする設計。

## 参考
- Tailwind Preflight: `ol, ul { list-style: none; margin: 0; padding: 0; }`
- 現状の `reset.css` は `*` で `margin: 0` をリセットしているが、`list-style` と `padding` はリセットしていない
- 影響: openground-ec のパンくずリスト（`ProductDetail.tsx`, `checkout/page.tsx`）で番号が表示される問題
